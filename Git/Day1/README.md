# Git Day 1 - Git Basics and Version Control

## Objective

The objective of Day 1 was to understand the fundamentals of Git, version control, repository creation, staging, committing, and tracking file changes.

---

## Topics Covered

- What is Git?
- What is Version Control?
- Git Repository
- Working Directory
- Staging Area
- Local Repository
- Tracked and Untracked Files
- Git Commit History

---

## Commands Practiced

- git init
- git status
- git add
- git commit
- git log
- git diff
- git diff --cached
- git restore --staged
- git rm --cached

---

## Git Workflow

```text
Working Directory
        │
        │ git add
        ▼
Staging Area
        │
        │ git commit
        ▼
Local Repository
```

---

## Concepts Learned

### Working Directory

The place where files are created and modified.

### Staging Area

Temporary area where Git stores changes before committing.

### Local Repository

The place where Git permanently stores commits on the local computer.

### Commit

A snapshot of the project at a specific point in time.

---

## Learning Outcome

After completing Day 1, I can:

- Create a Git repository.
- Check repository status.
- Stage files.
- Commit changes.
- View commit history.
- Compare file changes.
- Understand tracked and untracked files.
